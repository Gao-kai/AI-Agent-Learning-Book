## 1. 自动求导模块核心API

### 1.1 requires_grad 与叶子节点

PyTorch的张量有一个关键属性 `requires_grad`，决定它是否参与梯度计算。

```python
import torch

# 方式一：创建时指定
w = torch.tensor([2.0], requires_grad=True)

# 方式二：事后修改
x = torch.tensor([3.0])
x.requires_grad = True
```

**叶子节点（Leaf Node）**：直接创建的、不依赖其他张量计算的张量。

```python
w = torch.tensor([2.0], requires_grad=True)  # 叶子节点
x = torch.tensor([3.0], requires_grad=True)  # 叶子节点

y = w * x   # 非叶子节点（由其他张量计算而来）
loss = y ** 2  # 非叶子节点

print(w.is_leaf)      # True
print(x.is_leaf)      # True
print(y.is_leaf)      # False
print(loss.is_leaf)   # False
```

**核心特性对比**：

| 属性 | 叶子节点 (w, x) | 非叶子节点 (y, loss) |
|------|-----------------|----------------------|
| `is_leaf` | True | False |
| `grad` 属性 | **会保留梯度** | 默认不保留 |
| 内存效率 | 节省内存 | 释放后不留存 |

**计算图结构示例**：

```
w (叶子, requires_grad=True) ──→ multiply ──→ y (非叶子) ──→ power ──→ loss (非叶子)
x (叶子, requires_grad=True) ──→ ↗                                      ↓
                                                                         ↓
                                                            backward() 反向传播
                                                                         ↓
                                                      w.grad ←──────────┘
```

### 1.2 w.data 与 w 的本质区别（重点！）

**核心问题**：`w` 和 `w.data` 都是张量，它们有什么区别？

```python
import torch

w = torch.tensor([2.0], requires_grad=True)

print(w)                    # tensor([2.], requires_grad=True)
print(w.data)               # tensor([2.])  ← 没有grad信息！
print(w.requires_grad)      # True
print(w.data.requires_grad) # False ← 关键区别！
```

**核心区别表格**：

| 特性 | `w` | `w.data` |
|------|-----|----------|
| 计算图 | 在图中 | **脱离**计算图 |
| 梯度追踪 | ✅ 是 | ❌ 否 |
| .grad 属性 | 有 | 无（永远是 None） |
| 用途 | 正常训练流程 | 手动修改参数值（不追踪梯度） |

**底层原理图解**：

```
w 的内部结构：
┌─────────────────────────────────────────────────────────┐
│  torch.Tensor 对象 (w)                                   │
│  ├── data 属性 ────────────────▶ [底层数据 100.0]       │
│  ├── requires_grad = True                               │
│  ├── grad = tensor([200.])  ← 梯度存在这里               │
│  └── grad_fn = None (叶子节点没有 grad_fn)               │
└─────────────────────────────────────────────────────────┘

w.data（脱离计算图）：
┌─────────────────────────────────────────────────────────┐
│  w.data (只是一个普通张量)                                │
│  ├── 不在计算图中                                        │
│  ├── 没有 requires_grad 信息                            │
│  ├── 没有 grad 或 grad_fn                               │
│  └── 只是指向 w 底层数据的引用                           │
└─────────────────────────────────────────────────────────┘
```

**关键理解**：修改 `w.data` 就是修改 `w` 内部的数据，因为它们指向**同一块底层数据**！

```python
# 验证：w 和 w.data 共享同一份数据
import torch

w = torch.tensor([100.0], requires_grad=True)

print(f"更新前 w = {w.item()}")        # 100.0
print(f"更新前 w.data = {w.data.item()}")  # 100.0

# 通过 w.data 更新
w.data = w.data - 0.01 * torch.tensor([200.])

print(f"更新后 w = {w.item()}")        # 98.0  ← w 也变了！
print(f"更新后 w.data = {w.data.item()}")  # 98.0
```

**为什么要用 w.data 更新参数？**

```python
# ❌ 错误：直接赋值会创建新张量，断开计算图
# w = w - 0.01 * w.grad  # 这会让 w 指向新的张量，断开与计算图的连接

# ✅ 正确：修改底层数据不断开计算图
w.data = w.data - 0.01 * w.grad
```

### 1.3 核心API速查表

| API | 作用 |
|-----|------|
| `tensor.requires_grad=True` | 标记参与梯度计算 |
| `tensor.backward()` | 反向传播计算梯度 |
| `tensor.grad` | 获取梯度值 |
| `tensor.grad.zero_()` | 清零梯度（累加模式下必须） |
| `tensor.data` | 获取底层数据（脱离计算图） |
| `torch.no_grad()` | 临时禁用梯度追踪 |
| `retain_graph=True` | 保留计算图供多次backward |

## 2. 说清楚Loss.backward()的过程（重点！）

### 2.1 什么是反向传播？

反向传播（Backpropagation）是深度学习中最核心的算法，用于计算损失函数对每个参数的梯度。

**数学基础：链式法则**

如果 $L = f(g(x))$，则 $\frac{dL}{dx} = \frac{dL}{dg} \cdot \frac{dg}{dx}$

### 2.2 backward() 内部执行步骤

以 `Loss = w² + 20` 为例，调用 `Loss.backward()` 时，PyTorch 内部做了以下工作：

**Step 1: 从当前张量反向遍历计算图**

```
Loss = w² + 20
         ↓
    ┌────┴────┐
    │         │
    w²       20
    │
    w
```

**Step 2: 链式法则计算梯度**

```python
dLoss/dw = d(w² + 20)/dw = 2w
```

**Step 3: 梯度存储到叶子节点**

```python
w = torch.tensor([100.0], requires_grad=True)
Loss = w ** 2 + 20  # Loss = 10020

Loss.backward()  # 计算 dLoss/dw = 2w = 200

print(w.grad)    # tensor([200.])
print(Loss)      # tensor([10020.]) ← Loss本身不变！
```

### 2.3 计算图构建与反向传播完整图解

```
【Forward 正向传播】构建计算图         【Backward 反向传播】计算梯度

w = 100                                      ▲
    │                                         │
    ▼                                         │
w² = 10000                                    │
    │                                         │
    ▼                                         │
w² + 20 = 10020 = Loss ──────────────────────┘
                            dLoss/dw = 2w = 200
                            梯度从 Loss 反向传回 w
```

**对于更复杂的例子**：

```python
x = torch.tensor([2.0], requires_grad=True)
w = torch.tensor([3.0], requires_grad=True)
b = torch.tensor([1.0], requires_grad=True)

y = w * x + b          # y = 3*2 + 1 = 7
loss = (y - 5) ** 2    # loss = (7-5)² = 4
```

```
Forward 方向（构建图）:
x → w → multiply → add → y → subtract → power → loss
                    ↑
                    b

Backward 方向（计算梯度）:
loss → power → subtract → y → add → multiply → x
  ↓                                        ↓
  1.0    ←  dL/dy  ←  dL/dy  ←  dL/dy  ←  dL/dx (存储到 x.grad)
                                          ↑
                                    w.grad (存储到这里)
```

### 2.4 backward() 的三大要点（必须记住！）

1. **只计算梯度，不改变值**：Loss.backward() 不会修改任何张量的值
2. **沿着计算图反向传播**：从输出节点追溯到所有叶子节点
3. **梯度累加到 .grad**：每次调用会累积（+=）而不是覆盖（=）

```python
# 验证：backward() 不改变 Loss 的值
w = torch.tensor([2.0], requires_grad=True)
Loss = w ** 2 + 10

print(f"反向传播前 Loss = {Loss.item()}")  # Loss = 14.0
Loss.backward()
print(f"反向传播后 Loss = {Loss.item()}")  # Loss = 14.0 (没变！)
print(f"w.grad = {w.grad.item()}")          # w.grad = 4.0 (这是新计算的)
```

## 3. 为什么需要梯度清零（重点！）

### 3.1 PyTorch 默认梯度累加

这是 PyTorch 与其他框架的一个关键区别：**PyTorch 中梯度的默认行为是累加（+=），而不是覆盖（=）！**

```python
import torch

w = torch.tensor([1.0], requires_grad=True)

# 第一次backward
loss1 = w * 2
loss1.backward()
print(w.grad)  # tensor([2.])

# 第二次backward（不清零！）
loss2 = w * 3
loss2.backward()
print(w.grad)  # tensor([5.])  ← 2 + 3 = 5，累加了！
```

### 3.2 图解梯度累加机制

```
【第一次 backward】

loss1 = w * 2 = 2
backward()
    ↓
w.grad = 2

【第二次 backward（不清零）】

loss2 = w * 3 = 3
backward()
    ↓
w.grad = 2 + 3 = 5  ← 累加！
```

### 3.3 为什么需要清零梯度？（核心问题）

**标准训练循环**：

```python
for i in range(100):
    Loss = w ** 2 + 20          # 重新计算Loss（因为w变了）
    w.grad.zero_()               # ← 必须清零！
    Loss.backward()              # 计算梯度
    w.data = w.data - 0.01 * w.grad  # 更新权重
```

**如果不清零，会发生什么**：

```
第1次循环：w.grad = 200
第2次循环：w.grad = 200 + 196 = 396  ← 错误！应该是196
第3次循环：w.grad = 396 + 192 = 588  ← 越来越错！
梯度越来越大，完全错误！
```

**正确清零后**：

```
第1次循环：w.grad = 200（清零后计算）
第2次循环：w.grad = 196（清零后计算）
第3次循环：w.grad = 192（清零后计算）
每次都是正确的当前梯度！
```

### 3.4 PyTorch 为什么默认累加而不是覆盖？

**原因一：支持梯度累积技术（Gradient Accumulation）**

当 GPU 显存不够时，可以分批计算再累加：

```python
effective_batch_size = 64
mini_batch_size = 16
accumulation_steps = 4  # 64 / 16

model.zero_grad()
for i, (inputs, targets) in enumerate(dataloader):
    outputs = model(inputs)
    loss = criterion(outputs, targets) / accumulation_steps  # 归一化！
    loss.backward()

    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        model.zero_grad()
```

**数学上等价于使用大批量**：

```python
# 直接使用大批量
loss_big = sum(loss1, loss2, loss3, loss4)
loss_big.backward()
# dL/dw = dL1/dw + dL2/dw + dL3/dw + dL4/dw

# 分批计算 + 累加（效果相同）
loss1.backward()      # dL1/dw
loss2.backward()      # dL1/dw + dL2/dw
loss3.backward()      # dL1/dw + dL2/dw + dL3/dw
loss4.backward()      # dL1/dw + dL2/dw + dL3/dw + dL4/dw
```

**原因二：提供更大的灵活性**

```python
# 场景：你想手动控制何时清零
for epoch in range(10):
    for batch in data_loader:
        outputs = model(batch)
        loss = criterion(outputs, labels)
        loss.backward()

        # 可以选择每几步更新一次，而不是每步都更新
        if some_condition:
            optimizer.step()
            model.zero_grad()  # 累积了多步后才清零
```

### 3.5 清零梯度的三种方式

```python
# 方式一：使用 optimizer（推荐）
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)
optimizer.zero_grad()

# 方式二：直接对 grad 清零
w.grad.zero_()

# 方式三：对所有参数清零
for param in model.parameters():
    if param.grad is not None:
        param.grad.zero_()
```

## 4. 为什么循环内要重新计算 Loss（重点！）

### 4.1 核心问题

看这段代码：

```python
for i in range(1, 101):
    Loss = w ** 2 + 20      # ← 为什么每次都要重新计算？
    w.grad.zero_()
    Loss.backward()
    w.data = w.data - 0.01 * w.grad
```

**答案**：因为 `w.data` 更新的是 `w` 的底层数据，而 `Loss = w ** 2 + 20` 是一个数学表达式，**w 变了，Loss 的值就必须重新计算**！

### 4.2 图解整个训练循环过程

```
初始状态：
w = 100

第 1 次循环：
┌─────────────────────────────────────────────────────────┐
│ w = 100                                                 │
│ Loss = w² + 20 = 10000 + 20 = 10020  ← 第一次计算 Loss  │
│ dLoss/dw = 2w = 200                                    │
│ w = w - 0.01 * 200 = 100 - 2 = 98      ← w 更新了！    │
└─────────────────────────────────────────────────────────┘

第 2 次循环：
┌─────────────────────────────────────────────────────────┐
│ w = 98  ← 上一次更新后的值，不是 100 了！               │
│ Loss = w² + 20 = 9604 + 20 = 9624   ← 必须重新计算！   │
│ dLoss/dw = 2w = 196                                    │
│ w = w - 0.01 * 196 = 98 - 1.96 = 96.04  ← w 又更新了！ │
└─────────────────────────────────────────────────────────┘
```

### 4.3 w.data 和 w 的关系（再次强调）

**关键理解**：`w.data` 和 `w` 指向**同一块底层数据**，修改 `w.data` 就是修改 `w`：

```python
import torch

w = torch.tensor([100.0], requires_grad=True)

print(f"更新前 w = {w.item()}")        # 100.0
print(f"更新前 w.data = {w.data.item()}")  # 100.0

# 执行 w.data = w.data - 0.01 * w.grad
w.grad = torch.tensor([200.])
w.data = w.data - 0.01 * w.grad  # w.data = 100 - 2 = 98

# 结果：w 和 w.data 同时改变！
print(f"更新后 w = {w.item()}")        # 98.0
print(f"更新后 w.data = {w.data.item()}")  # 98.0
```

### 4.4 训练循环的标准流程

```python
for i in range(100):
    # ========== 正向传播 ==========
    # 必须重新计算，因为 w 已经变了
    Loss = w ** 2 + 20

    # ========== 反向传播 ==========
    # 清零上次的梯度（否则会累加）
    w.grad.zero_()
    # 计算 dLoss/dw
    Loss.backward()

    # ========== 更新权重 ==========
    # 更新 w，但不改变 Loss
    w.data = w.data - 0.01 * w.grad

    # 此时 Loss 仍然是旧值，循环回去重新计算新的 Loss
```

## 5. 正向传播和反向传播的区别

### 5.1 定义对比

| 方面 | 正向传播 (Forward) | 反向传播 (Backward) |
|------|-------------------|-------------------|
| 方向 | 从输入到输出 → | 从输出到输入 ← |
| 作用 | 计算预测值 $\hat{y}$ | 计算梯度 $\frac{\partial L}{\partial w}$ |
| 触发 | `output = model(input)` | `loss.backward()` |
| 记忆 | 构建计算图 | 释放计算图 |

### 5.2 图解完整流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        训练循环                                  │
│                                                                 │
│  ┌───────────────┐                                             │
│  │ 正向传播        │  w = 100                                   │
│  │ Forward Pass   │  Loss = w² + 20 = 10020                    │
│  │                │  构建计算图                                 │
│  └───────┬───────┘                                             │
│          │                                                     │
│          ▼                                                     │
│  ┌───────────────┐                                             │
│  │ 计算Loss      │  比较预测值与真实值                          │
│  └───────┬───────┘                                             │
│          │                                                     │
│          ▼                                                     │
│  ┌───────────────┐                                             │
│  │ 反向传播       │  dLoss/dw = 2w = 200                       │
│  │ Backward Pass │  梯度存储在 w.grad                          │
│  └───────┬───────┘                                             │
│          │                                                     │
│          ▼                                                     │
│  ┌───────────────┐                                             │
│  │ 更新权重       │  w = w - lr * w.grad = 100 - 0.01*200 = 98 │
│  └───────┬───────┘                                             │
│          │                                                     │
│          └─────────────▶ 下一轮循环                             │
└─────────────────────────────────────────────────────────────────┘
```

### 5.3 代码对应关系

```python
# ==================== 正向传播 ====================
# 输入 → ... → 输出
x = torch.tensor([2.0], requires_grad=True)
w = torch.tensor([3.0], requires_grad=True)
b = torch.tensor([1.0], requires_grad=True)

y = w * x + b          # Forward: 计算 y = 3*2 + 1 = 7
loss = (y - 5) ** 2    # Forward: 计算 loss = (7-5)² = 4

# ==================== 反向传播 ====================
# 输出 ← ... ← 输入
loss.backward()        # Backward: 计算 dLoss/dw, dLoss/db, dLoss/dx

print(w.grad)          # dLoss/dw = 2*(y-5)*x = 2*2*2 = 8
print(b.grad)          # dLoss/db = 2*(y-5)*1 = 2*2 = 4
print(x.grad)          # dLoss/dx = 2*(y-5)*w = 2*2*3 = 12
```

### 5.4 数学角度的理解

**正向传播**：计算 $L = f(x, w)$

```
输入 x + 权重 w
    │
    ▼
线性变换: z = wx + b
    │
    ▼
激活函数: a = σ(z)
    │
    ▼
输出预测 ŷ
```

**反向传播**：计算 $\frac{\partial L}{\partial w}$（链式法则）

```
损失 L = (ŷ - y)²
    ▲
    │
梯度反向传播
    ▲
    │
dL/dŷ → dŷ/da → da/dz → dz/dw
    │
    ▼
∂L/∂w = ∂L/∂ŷ × ∂ŷ/∂a × ∂a/∂z × ∂z/∂w
```

## 6. 实战：完整的训练循环

```python
import torch

# 1. 定义参数 w，设置 requires_grad=True 让 PyTorch 追踪梯度
w = torch.tensor([100.0], requires_grad=True)
learning_rate = 0.01

print("=" * 60)
print("初始状态")
print(f"w = {w.item()}")
print("=" * 60)

# 2. 训练循环
for i in range(1, 6):
    # 【正向传播】计算当前 w 下的 Loss
    Loss = w ** 2 + 20

    print(f"\n第 {i} 次迭代")
    print(f"  正向传播: Loss = w² + 20 = {w.item():.2f}² + 20 = {Loss.item():.2f}")

    # 【梯度清零】防止累加
    if w.grad is not None:
        w.grad.zero_()

    # 【反向传播】计算 dLoss/dw
    Loss.backward()

    print(f"  反向传播: dLoss/dw = 2w = {w.grad.item():.2f}")

    # 【更新权重】w_new = w_old - lr * grad
    w.data = w.data - learning_rate * w.grad

    print(f"  更新权重: w = {w.item():.2f} - {learning_rate} × {w.grad.item():.2f} = {w.item():.2f}")

print("\n" + "=" * 60)
print(f"最终结果: w = {w.item():.4f}, Loss = {w.item()**2 + 20:.4f}")
print(f"理论最优解: w = 0, Loss = 20")
print("=" * 60)
```

**运行结果**：

```
============================================================
初始状态
w = 100.0
============================================================

第 1 次迭代
  正向传播: Loss = w² + 20 = 100.00² + 20 = 10020.00
  反向传播: dLoss/dw = 2w = 200.00
  更新权重: w = 100.00 - 0.01 × 200.00 = 98.00

第 2 次迭代
  正向传播: Loss = w² + 20 = 98.00² + 20 = 9624.00
  反向传播: dLoss/dw = 2w = 196.00
  更新权重: w = 98.00 - 0.01 × 196.00 = 96.04

第 3 次迭代
  正向传播: Loss = w² + 20 = 96.04² + 20 = 9255.68
  反向传播: dLoss/dw = 2w = 192.08
  更新权重: w = 96.04 - 0.01 × 192.08 = 94.12

...

============================================================
最终结果: w = 80.0000, Loss = 6020.0000
理论最优解: w = 0, Loss = 20
============================================================
```

## 总结

| 概念 | 核心要点 |
|------|---------|
| **requires_grad** | 标记张量是否参与梯度计算 |
| **叶子节点** | 直接创建的张量（is_leaf=True），梯度会保留 |
| **w.data** | 脱离计算图的底层数据引用，修改它就是修改 w |
| **backward()** | 只计算梯度，不改变任何张量的值 |
| **梯度清零** | PyTorch默认梯度累加（+=），必须清零，否则梯度错误 |
| **正向传播** | 计算预测值，构建计算图 |
| **反向传播** | 沿计算图反向计算梯度，存储到叶子节点的 .grad |
| **Loss 重新计算** | 因为 w 通过 w.data 更新后变了，Loss 必须重新算 |

**最重要的三个理解**：

1. **`w.data` 和 `w` 是同一块数据**：修改 `w.data` 就是修改 `w`
2. **`backward()` 只算梯度不改值**：Loss 的值在 backward 前后不变
3. **梯度默认累加**：不清零会导致梯度错误累积
